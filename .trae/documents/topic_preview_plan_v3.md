# 考点题目预览功能实现计划

## 1. 项目现状分析

### 1.1 系统结构
- **前端**：HTML5 + CSS3 + JavaScript，使用Font Awesome图标库
- **后端**：Python + Flask框架
- **数据存储**：文件系统（Markdown题目文件、JSON配置文件）
- **依赖**：Flask, Flask-CORS, pypandoc (可选，用于生成Word文档)

### 1.2 关键文件
- `frontend/index.html`：前端主页面
- `frontend/js/main.js`：前端主要逻辑
- `frontend/css/style.css`：前端样式
- `backend/app.py`：后端主文件
- `backend/paper_generator.py`：试卷生成核心逻辑
- `backend/question_parser.py`：题目解析器

## 2. 功能需求

1. **鼠标样式变化**：当鼠标移动到题库目录树最下层的每个考点区域时，鼠标由箭头变成抓手状
2. **点击展开/收起**：点击考点区域，展开相应的考点题目；再次点击，收起题目
3. **题目渲染**：使用texme渲染每一个题目，一个题目一条记录
4. **只展开题目**：不要展开答案和答案解析部分
5. **位置要求**：就在题目目录树下面一次展开并渲染显示，不要开辟另外的预览区域
6. **texme使用**：只渲染考点里面的每一个题目，避免texme默认将第一条渲染为标题
7. **保持其他功能**：只增加这一个功能，其它的都不要修改，保存和原来一样

## 4. 技术实现细节

### 4.1 texme使用
- 使用CDN加载texme库：`https://cdn.jsdelivr.net/npm/texme@1.2.2`
- 配置texme：`window.texme = { style: 'plain', renderOnLoad: false }`
- 使用 `window.texme.render(content)` 方法渲染题目内容
- 为了避免texme默认将第一条渲染为标题，可以在题目内容前添加一个空行，或者使用texme的配置选项

### 4.2 题目预览实现
- 点击考点时，发送请求到 `/api/bank/questions` 端点，获取题目数据
- 渲染题目预览，最多显示3道题
- 超过3道题时，显示"更多"按钮，点击后显示所有题目
- 使用texme渲染每道题目，确保数学公式正确显示
- 点击"查看详情"按钮，显示题目详情，包括答案和解析

### 4.3 事件绑定
- 使用 `addEventListener` 绑定考点点击事件
- 使用 `e.stopPropagation()` 防止事件冒泡
- 使用 `async/await` 处理异步请求

## 5. 风险处理

### 5.1 潜在问题
- texme库与系统其他部分的冲突
- 题目渲染失败
- 异步请求超时
- 页面布局混乱

### 5.2 解决方案
- 配置texme，禁用自动渲染，只在需要时渲染题目
- 添加错误处理，显示加载失败的提示
- 添加加载状态，提升用户体验
- 确保CSS样式正确，避免页面布局混乱
