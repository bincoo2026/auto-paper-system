# 修改编辑框初始内容设置方式为insertContent

## 问题描述

当前编辑题目对话框使用`setContent`方法设置编辑器初始内容，这可能导致@aarkue/tiptap-math-extension扩展无法正确激活数学公式的渲染。需要改为使用`insertContent`方法来确保数学公式能够被正确识别和渲染。

## 计划步骤

### 步骤1：学习Tiptap的insertContent使用方法
- 查阅Tiptap官方文档，了解`insertContent`方法的语法和参数
- 了解如何通过`insertContent`插入不同类型的内容，包括普通文本和数学公式

### 步骤2：分析当前代码结构
- 查看`openEditModal`和`openAddQuestionModal`方法中设置编辑器内容的代码
- 理解当前如何处理`stemContent`、`answerContent`和`analysisContent`

### 步骤3：创建解析函数
- 创建`insertParsedContent`函数，用于解析输入字符串并分段插入内容
- 函数需要：
  1. 识别字符串中的普通文本部分
  2. 识别行内公式（`$...$`）
  3. 识别块级公式（`$$...$$`）

### 步骤4：实现解析逻辑
- 使用正则表达式或字符串操作来分割内容
- 对于普通文本：调用`editor.commands.insertContent('文本')`
- 对于行内公式：调用`editor.commands.insertContent({ type: 'inlineMath', attrs: { latex: '公式内容', display: 'no' } })`
- 对于块级公式：调用`editor.commands.insertContent({ type: 'inlineMath', attrs: { latex: '公式内容', display: 'yes' } })`

### 步骤5：修改现有代码
- 在`openEditModal`方法中，将：
  ```javascript
  editors.stemEditor.commands.setContent(stemContent);
  editors.answerEditor.commands.setContent(answerWithBreaks);
  editors.analysisEditor.commands.setContent(analysisWithBreaks);
  ```
  替换为：
  ```javascript
  this.insertParsedContent(editors.stemEditor, stemContent);
  this.insertParsedContent(editors.answerEditor, answerWithBreaks);
  this.insertParsedContent(editors.analysisEditor, analysisWithBreaks);
  ```

- 在`openAddQuestionModal`方法中，保持清空编辑器的逻辑不变

### 步骤6：测试验证
- 测试块级公式`$$x^2+y^2=z^2$$`是否正确渲染
- 测试行内公式`$a+b$`是否正确渲染
- 测试普通文本是否正确显示
- 测试混合内容（文本+公式）是否正确显示

## 技术实现细节

### 解析算法思路
1. 使用正则表达式分割字符串，识别`$$...$$`和`$...$`模式
2. 按照分割后的顺序，依次处理每个部分
3. 对于公式部分，提取公式内容并创建相应的insertContent调用

### 可能的挑战
- 处理嵌套公式的情况
- 处理转义字符（如`\$`）
- 确保性能，避免处理大文本时的延迟
