# 删除章文件夹功能实现计划

## 功能需求
1. 在前端章题库题型下面的章文件夹右侧 重命名 图标按钮的后面，新增考点 图标按钮的前面，增加 删除 图标按钮
2. 点击删除图标按钮，弹出是否确认删除的消息框
3. 若点击 确认，则从后端 题库 对应的题型文件夹删除 对应的章目录文件夹
4. 删除完成后，重新加载科目题库，并切换回当前页面刷新显示

## 实现步骤

### 前端部分

#### 步骤1：添加删除按钮到章文件夹
- 文件：`/home/zhao/project/auto-paper-system/frontend/js/main.js`
- 位置：在 `renderBank` 方法中，章文件夹的 HTML 生成部分
- 在重命名按钮后面，新增考点按钮前面添加删除按钮
- 按钮样式参考删除考点按钮

```javascript
// 章文件夹删除按钮
<span class="delete-chapter-badge" title="删除章文件夹" onclick="event.stopPropagation(); paperComposer.openDeleteChapterModal('${key}')">
    <i class="fas fa-trash-alt"></i>
</span>
```

#### 步骤2：添加删除确认模态框
- 文件：`/home/zhao/project/auto-paper-system/frontend/index.html`
- 在 `delete-topic-modal` 后面添加 `delete-chapter-modal`
- 模态框内容与删除考点类似

#### 步骤3：添加前端方法
- 文件：`/home/zhao/project/auto-paper-system/frontend/js/main.js`
- 添加以下方法：
  - `openDeleteChapterModal(key)` - 打开删除章文件夹对话框
  - `closeDeleteChapterModal()` - 关闭删除章文件夹对话框
  - `saveDeleteChapter()` - 保存删除章文件夹操作

#### 步骤4：实现删除逻辑
- 在 `saveDeleteChapter()` 方法中：
  - 解析章文件夹路径
  - 发送 POST 请求到 `/api/bank/delete-chapter`
  - 成功后重新加载题库结构
  - 关闭对话框并显示成功消息

### 后端部分

#### 步骤5：添加后端路由
- 文件：`/home/zhao/project/auto-paper-system/backend/app.py`
- 添加 `/api/bank/delete-chapter` POST 路由
- 参考 `delete_topic` 函数的实现

#### 步骤6：实现删除章文件夹逻辑
- 在 `delete_chapter()` 函数中：
  - 解析 `chapter_key` 获取路径信息
  - 构建章文件夹的完整路径
  - 检查文件夹是否存在
  - 使用 `shutil.rmtree()` 删除文件夹
  - 清除相关缓存
  - 返回成功响应

## 技术要点

1. **路径解析**：参考删除考点的逻辑，从 `chapter_key` 解析出科目、题型、章名称
2. **权限控制**：确保用户只能删除自己的章文件夹
3. **错误处理**：处理路径无效、文件夹不存在等异常情况
4. **缓存清理**：删除后清除相关缓存，确保下次加载的是最新数据
5. **UI一致性**：删除按钮样式与其他删除按钮保持一致
